# 以下介绍各个标签
## html
 * 对于xhtml页面来说，根元素可以是其它标签
 * 跟标签 root element
 * 一个页面只能有一个
 * 没有的话。浏览器也会自动添加
## head
  * 头部标签,放一些与页面相关的"元信息",比如:
    * 页面的编码方式
      - '<mate charset="utf-8">'
        + chartset:字符集
    * 页面图标可以用link标签来放置
## body
  * 想要在页面中被显示出来的内容都放在这个标签里面
  * 如果页面没有html或者body标签,浏览器会自动添加,并将页面内容包含在body里面
    * 但浏览器同时也会将那些必须放在head里的标签放进head里面,比如title标签等

    ```html
    <title>页面标题</title>
    <h1>一级标题</h1>
    <p>一个段落</p>
    ```

    * 上面的代码会被浏览器解析为下面的结构

    ```html
    <!doctype html> 这一行不会自动添加
    <html>
      <head>
        <title>
          页面标题
        </title>
      </head>
      <body>
        <title>页面标题</title>
        <h1>一级标题</h1>
        <p>一个段落</p>
      </body>
    </html>
    ```
    * 同时,如果在body或者html标签的**结束标签之后**又出现了其它的标签,则之前的结束标签就会被认为无效,浏览器会自动添加结束标签
## base 基础
  * `<base href="页面中所有相对路径的基准地址" target="页面中所有链接的打开位置">`
    * 一定要以/即目录结尾
      - `<base href="https://www.baidu.com/abc/">`
      - `<base href="https://www.baidu.com/abc">`相当于`<base href="https://www.baidu.com">`
    * target属性表示页面中**所有链接**的打开位置
      -当然,可以在页面内的a标签中用它自己的target属性覆盖用base标签设置的全局打开位置,也就是页内标签的优先级高
  * 有些框架可能会利用这个特性
## h1到h6 标签
  * 默认情况下,标题上下会有一定的空白
  * 比较传统的观点认为一个页面不能出现超过一个h1标签,原因是为了seo搜索引擎
    - seo是什么
      + 搜索   引擎   优化
        * 讲白了就是如何布置页面能让页面在搜索结果中尽量靠前
        * 手段有很多种
          + 如关键字
          + 页面被引用的次数(反向链接数量)
            * 交换链接
          + 使用https
          + 使页面的html更符合语义
  * HTML5中新增了hgrooup (即header group) 标签,大致用法如下: H5

  ```html
  <hgroup>
    <h2>三体</h2>
  </hgroup>
  ```
    + 即当不同级别标题显示在一起的时候可以用这个标签吧Hn标签归类
    + 一般来说,这个标签可以替换以前使用div的场景
## P 标签
  * paragraph
  * 语意是一个段落,当想要表达一段话的时候可以使用这个标签
  * 默认情况下也会有上下边距
  * 块级元素
## a 标签
  * anchor,锚
  * 语意是一个链接,链接地址写在 `href`(hyperlink reference)属性里
    + 可以链接到的类型非常多,而且一般来说是可扩展的
      - 绝对地址,fullpath
        * https://jd.com
      - 页内特定位置跳转地址
        * #pos1
      - 其他页面特定位置跳转地址
        * https://jd.com/#footer
      - 这个在书目的章节跳转使用比较多
      - 相对路径
      - 电子邮件
      - 电话号码
      - QQ/TAOBAO 临时会话
      - 空白的href属性会链接到当前页面(注意base属性)
  * target属性
    + 可以指定在哪个窗口打开链接
      - 几个特殊值,关键字
        * _blank,链接在空白的窗口显示,也就相当于新打开一个标签
        * _self,默认值,在当前窗口打开
        * _parent,链接在父窗体显示
        * _top,链接在顶层窗体显示
  * 在html5中,还有一个download属性
    + 表示点击这个链接将下载链接对应的文件,而不是跳转到该目标页面,下载的文件名以download属性的值来命名 (下载内容应该是一个相对路径)
      - 如果同时有target="_blank",又有download属性,它如何行为呢?
    + 为什么要有这个属性呢?传统浏览器中,要触发下载,需要服务端的支持,给出特定的http头太会出发浏览器下载而不是打开对应的内容
      - 这个属性的出现可以让点击下载完全由前端来完成
## img 标签
  * image
  * 表示一张图片
  * 用src(source)属性来表示图片的地址
    + 同样也可以是绝对地址,相对地址
    + 可以简单了解一下`srcset`属性
    ```html 
    <img srcset="a.jpg 1x, b.jpg 2x, c.jpg 3x">
    ```
  * src指定的图片可以是浏览器支持的任何图片格式
    - jpg/jpeg,png,bmp,gif,webp,svg
    - 注意psd格式是不支持的,因为它是Photoshop专有的格式,支持它需要大量的代码而且psd格式的文件一i版很大,一般都在几十M上下
  * 同时,用alt属性表示图片在加载失败时的替换文本
    - 类似于markdown中图片的替换文档`[加载失败的替换文本](image source)`,事实上两者极本上是等同的
    - alt的全称是alternative
      + 意思是替代
      + 键盘上的alt键是这个单词的前缀,即alter
  * 可以使用width和height两个属性定义图片的宽和高
    - 只写一个的话,另一个会根据图片原始尺寸的比例计算出来
    - 图片一般来说有自己的宽高
    - 但是图片加载往往需要时间,而图片在加载完成之前的浏览器是不知道其宽高的,所以就会产生页面抖动的问题
    - 所以一般会在标签上把宽高写出来,这样图片加载过程中页面就不会抖动了
    -还有需要注意的一点`图片上的宽和高是不需要加单位的,直接以数字表示`
## span 标签
  * 是一个没有明确语意(通用语意)的标签
  * 一般来说想要给特定的内容加样式时可以用一个span标签将内容包起来
